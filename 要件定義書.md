## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| ジャンル | トップダウン型ローグライトアクション（アーチャー伝説ライク） |
| プラットフォーム | ブラウザ（PC・スマホ両対応） |
| デプロイ先 | Vercel |
| 開発環境 | VSCode + Claude |
| 開発言語 | TypeScript |

### コンセプト

- プレイヤーは移動中に攻撃できない。**止まると自動で最も近い敵に攻撃する**
- 部屋をクリアするたびに**スキルを1つ選択**し、キャラクターを強化する
- **スマホファースト**。バーチャルパッドのみで完結する操作体系

## 3. 操作仕様

### 移動

| 項目 | 仕様 |
|------|------|
| 操作方法 | **バーチャルパッドのみ** |
| キーボード操作 | **なし** |
| スティックの傾き量 | 移動速度に比例（最大値でキャップ） |
| PC対応 | マウスドラッグでパッドを操作可能 |

### 攻撃

| 項目 | 仕様 |
|------|------|
| 攻撃条件 | プレイヤーが**停止中**かつ画面内に敵がいる |
| 攻撃操作 | **自動**（プレイヤーは操作不要） |
| ターゲット | 最も近い敵 |

### バーチャルパッド詳細

| 項目 | 仕様 |
|------|------|
| 配置 | 画面左下・固定 |
| 形状 | 外側リング + 内側スティック |
| 外側リング直径 | 120px |
| 内側スティック直径 | 50px |
| 動作 | タッチ/クリック開始位置を中心に追従 |
| スティック可動範囲 | 外側リングの端まで（半径60px） |
| イベント（スマホ） | touchstart / touchmove / touchend |
| イベント（PC） | mousedown / mousemove / mouseup |

## 4. ゲームフロー

```
[タイトル画面]
    ↓ 画面タップ/クリック
[ゲーム画面] 部屋1
    ↓ 敵を全滅
[スキル選択画面]
    ↓ スキルを1つ選択
[ゲーム画面] 部屋2
    ↓ 敵を全滅
[スキル選択画面]
    ↓ スキルを1つ選択
[ゲーム画面] 部屋3
    ↓ 敵を全滅
[クリア画面]
    ↓ タイトルへボタン
[タイトル画面]

※ いずれかの部屋でHPが0になった場合：
[ゲーム画面] → [ゲームオーバー画面]
    ↓ リトライボタン → [ゲーム画面] 部屋1（スキルリセット）
    ↓ タイトルへボタン → [タイトル画面]
```

| パラメータ | 値 |
|------------|----|
| HP | 30 |
| 移動速度 | 60 px/s |
| 接触ダメージ | 8 |
| 攻撃クールダウン | 1.0秒（同じ敵に連続で当たらないように） |
| 表示サイズ（MVP） | 28×28px（赤い矩形） |
| 当たり判定サイズ | 24×24px |
| 行動パターン | プレイヤーに向かってまっすぐ移動 |
| スポーン位置 | プレイヤーから150px以上離れたランダム位置 |

### 敵の行動詳細

- **移動**: 毎フレーム、プレイヤーの方向へ向き直し移動
- **攻撃**: プレイヤーと接触中、クールダウンが0になったらダメージ
- **死亡**: HP0でその場で消滅（MVP: エフェクトなし）

## 7. 攻撃仕様

### MVP: ストレートショット（1種類）

| パラメータ | 値 |
|------------|----|
| 発射条件 | プレイヤー停止中・敵が1体以上存在 |
| ターゲット選択 | 最も近い敵（毎回再計算） |
| 弾道 | 発射時点でのターゲット方向へ直線（追尾なし） |
| 貫通 | なし（敵1体に当たると消滅） |
| 射程 | 画面外に出たら消滅 |
| 表示サイズ（MVP） | 8×8px（黄色い矩形） |

## 8. スキル仕様

### スキル選択ルール

- 部屋クリアのたびに**毎回同じ3種類**のカードが表示される（MVPはランダムなし）
- プレイヤーが1枚選択すると即座に効果が適用される
- 同じスキルを複数回取得可能（重複効果あり）

### スキル一覧（MVP: 3種類）

| スキルID | 名前 | 効果 | 重複 |
|----------|------|------|------|
| `attack_speed_up` | 連射強化 | 攻撃間隔 × 0.8（20%短縮） | 可（最小0.2秒） |
| `damage_up` | 攻撃力強化 | 弾ダメージ +5 | 可 |
| `hp_recover` | HP回復 | 現在HP +30（最大HP上限） | 可（上限超え不可） |

## 13. 拡張性設計

### 設計思想

**データ駆動アーキテクチャ**を採用する。  
ゲームロジックとデータ定義を分離し、`src/data/` 配下のファイルにエントリを追加するだけで新要素を追加できるようにする。

### 拡張方法早見表

| 追加したいもの | 作業内容 |
|----------------|---------|
| 新しい敵 | `src/data/enemies.ts` に定義を1つ追加 |
| 新しいスキル | `src/data/skills.ts` に定義を1つ追加 |
| 新しい攻撃方法 | `src/data/weapons.ts` に定義を1つ追加 |
| 新しいステージ | `src/data/stages.ts` に定義を1つ追加 |
| BGM追加 | `public/assets/audio/bgm/` に置いて `stages.ts` にパスを記載 |
| SE追加 | `public/assets/audio/se/` に置いて対応データに記載 |
| キャラ画像差し替え | `public/assets/images/` に置いてデータにパスを記載 |

### データ型定義（イメージ）

```typescript
// 敵定義の型
type EnemyDef = {
  id: string           // 識別子
  hp: number           // 最大HP
  speed: number        // 移動速度 px/s
  damage: number       // 接触ダメージ
  attackCooldown: number // 攻撃クールダウン（秒）
  width: number        // 表示幅
  height: number       // 表示高さ
  spriteKey?: string   // スプライトキー（未指定の場合はMVPの矩形）
  color?: number       // MVPの矩形カラー（16進数）
}

// スキル定義の型
type SkillDef = {
  id: string
  name: string
  description: string
  apply: (player: Player) => void  // 効果関数
}

// 攻撃定義の型
type WeaponDef = {
  id: string
  bulletSpeed: number
  damage: number
  fireInterval: number   // 発射間隔（秒）
  bulletWidth: number
  bulletHeight: number
  spriteKey?: string
  color?: number
  soundKey?: string      // SE
}

// ステージ定義の型
type StageDef = {
  id: string
  bgmKey?: string
  backgroundKey?: string
  backgroundColor?: number
  rooms: RoomDef[]
}

// 部屋定義の型
type RoomDef = {
  enemies: { enemyId: string; count: number }[]
}
```

## 16. スコープ外

以下はMVPに含まない。後から追加する可能性はある。

| 機能 | 備考 |
|------|------|
| セーブ機能 | ローカルストレージ等 |
| オンライン機能 | ランキング等 |
| ガチャ・課金 | 不要 |
| キーボード/マウスでの移動操作 | バーチャルパッドのみ |
| ボス戦 | MVPでは含まない |
| 複数ステージ | MVPは1ステージ3部屋 |
| アニメーション | MVPは矩形のみ |
| SE / BGM | MVPはなし |
| エフェクト（爆発・点滅等） | MVPはなし |
| スコア・ランキング | MVPはなし |
| チュートリアル | MVPはなし |
